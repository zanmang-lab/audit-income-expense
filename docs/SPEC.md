# 수입지출내역서 자동 작성 도구 — 빌드 명세서 (SPEC)

> 이 문서는 Cursor 바이브 코딩의 **기준 문서**입니다. 코드가 아래 규칙을 어기면 안 됩니다.
> 애매하거나 규칙에 없는 케이스는 **임의 추정 금지** → 해당 칸을 **공백**으로 두고 `검수필요` 플래그를 남깁니다.

---

## 0. 목표
- 입력: 엑셀 장부 1개 + 연체료 목록 + 환불자 목록 + (선택) 연체료 규정표
- 처리: 행을 페이지 단위로 묶고 각 칸을 규칙대로 채움
- 출력: **사업명별 한글 문서 1개**(여러 페이지 포함). 한글 양식 템플릿에 값만 채워 넣음.

---

## 1. 입력 파일 명세

### 1-1. 장부 (예: `261_학복위_엑셀장부_공유용.xlsx`, 시트 `DATA (2)`)
1행은 헤더, 2행부터 데이터. 한 행 = 한 세목.

| 열 | 헤더 | 용도 |
|---|---|---|
| A | 날짜 | 페이지 날짜 (`2026.0n.nn.` 형식으로 변환) |
| B | 사업번호 | 파일 분리 기준 |
| C | 세부번호 | 사업번호-세부번호 범위 표기에 사용 |
| D | 기재내용 | **세목** (절대 가공 금지, 그대로 복사) |
| E | 현금 | 사용 안 함 |
| F | 구분 | **그대로** 수입지출내역서 구분 칸에 복사. 비어 있으면 공백 + `검수필요` |
| G | 개요 | **내용** 칸 |
| H | 수입 | 수입 금액 |
| I | 지출 | 지출 금액 |
| J | 오류여부 | 값 있으면 `검수필요` 플래그 |

### 1-2. 연체료 목록 (`20261학기_연체료_목록.xlsx`)
컬럼: 반납일(입금일) / 대여물품 / 성명 / 연체료 / 비고
- 비고 값에 따라 분기: `필름값`, `손망실비`, `감면`, `OOO'으로 입금`(대납), 공백(=순수 연체)

### 1-3. 환불자 목록 (`환불자_목록.xlsx`)
컬럼: 사물함 번호 / 이름 / 전화번호 / 환불금액 / 사유 / 계좌번호
- **결과물에는 이름·사유만 사용.** 전화번호·계좌번호는 절대 출력 금지.
- 사유 종류: 당첨등록포기 / 두번송금 / 규칙미준수 / 중복등록포기 등

### 1-4. 연체료 규정표 — 사용자 제공 (필수 입력)
물품별 1일 단가. `config/학기설정.json`의 `overdue_rules`에 포함.
예: `{ "보조배터리": 1000, "우산": 2000, "농구공": 2000, "드릴": 2000, "듀라테이블(대)": 3000, "쇼핑카트": 3000, ... }`
- `연체료 ÷ 해당 물품 1일단가 = n일`로 자동 계산.
- 규정표에 없는 물품이거나 나눠떨어지지 않으면 → n일 공백 + `검수필요`.
- 필름값·손망실비·감면 등 연체가 아닌 항목은 규정표를 쓰지 않음(6-3 참조).

## 3. 페이지 분할 알고리즘 (핵심)
사업번호별로 행을 **장부 순서 그대로** 처리. 아래 키 중 **하나라도 직전 행과 다르면 새 페이지**.
**추가로, 한 페이지가 13줄(양식 세목 행 수)에 도달하면 키가 같아도 새 페이지로 끊는다.**

키 = (날짜, 수입/지출 여부, 장부 F열 구분)
MAX_ROWS_PER_PAGE = 13   # 양식 세목 행 수. 상수로 분리.

```
for 사업번호 in 장부:
    pages = []
    cur = None
    for row in 해당 사업번호 행들(원래 순서):
        key = (row.날짜, "수입" if row.수입>0 else "지출", row.구분)
        if cur is None or key != cur.key or len(cur.rows) >= MAX_ROWS_PER_PAGE:
            cur = new Page(key); pages.append(cur)
        cur.rows.append(row)
```
> - "수입 사이 지출 1건, 지출 사이 수입 1건도 무조건 페이지 변경"은 키의 *수입/지출 여부*로 자동 처리됨.
> - 13줄 상한 때문에 같은 날짜·구분이라도 `2-1~13`, `2-14~26`처럼 끊긴다(25-2 통합본과 동일).
> - 사업번호 범위 표기는 페이지 내 첫~끝 세부번호로 자동 계산.

## 4. 구분 (F열)
장부 F열 값을 **수정 없이** 수입지출내역서 구분 칸에 넣는다.
- 비어 있으면 → 구분 칸 **공백 + `검수필요`**
- 페이지 나누기 키에도 이 값을 사용한다 (날짜, 수입/지출, 구분)

~~구분 재계산 규칙 (역할 기반)~~ — 사용하지 않음. `src/classify.py`는 레거시 참고용.

## 5. 각 칸 채우기 규칙

| 칸 | 규칙 |
|---|---|
| 사업번호 | `사업번호-세부번호` 형식, 띄어쓰기 없음. 페이지에 여러 행이면 `사업번호-첫세부~끝세부` (예: `1-1~8`). 1건이면 `1-1`. |
| 사업명 | 2번 매핑값 |
| 날짜 | `2026.0n.nn.` (한 자리 월/일은 0 패딩). 날짜 바뀌면 이미 페이지가 분리되어 있음 |
| 수입/지출 | 페이지 키의 값 |
| 구분 | 장부 F열 값 그대로 |
| 내용 | 이 페이지가 무엇을 위한 것인지 한 줄. 장부 G열(개요)을 대표값으로 사용 |
| 세목 | 장부 D열 **그대로** (띄어쓰기·대소문자·오탈자까지 절대 수정 금지) |
| 수량 | 항상 `1` |
| 단가 | 수입 페이지면 H열, 지출 페이지면 I열 금액. **콤마 포함 표기** |
| 총액 | 페이지 내 단가 합계 (콤마 포함) |
| 증빙서류 | **공란** |
| 성명/직위/소속/연락처 | 양식 고정값 (작성자 정보) |
| 비고 | 6번 규칙 |

## 6. 비고 생성 규칙

### 6-1. 사물함 입금 (사업번호 1·2·9 등)
고정 문구:
```
전자 사물함은 대여비 4,000원, 일반 사물함은 대여비 2,000원
```

### 6-2. 사물함 환불
위 고정 문구 + 환불자별 한 줄:
```
OOO님은 [사유]로 환불 진행
```
- 환불자 목록 `이름`으로 매칭, `사유` 사용.
- 사유 정규화(선택): 당첨등록포기→"당첨 등록 포기", 두번송금→"중복 입금", 중복등록포기→"중복 등록 포기", 규칙미준수→"규칙 미준수".
- 이름 매칭 실패 → 그 줄 공백 + `검수필요`.

### 6-3. 물품 대여 (사업번호 3)
연체료 목록을 `성명`으로 매칭하고, 목록의 **비고 값**으로 분기:

| 목록 비고 | 출력 문구 |
|---|---|
| (공백, 순수 연체) | `OOO님 [대여물품] 대여 후 n일 연체되어 연체료 X원 부과` |
| `필름값` | `OOO님 폴라로이드 필름 m장 추가로 소모하여 X원 부과` (m = X÷1,000 가정, 확인 후) |
| `손망실비` | `OOO님 [대여물품] 손망실비 X원 부과` |
| `감면` | 금액만 반영, 사유는 공백 + `검수필요` |
| `OOO'으로 입금` | 대납 케이스 → 6-4 참조 |

- `n일`은 4번(연체료 규정표) 있을 때만 계산. 없으면 n 공백 + `검수필요`.
- X = 연체료 금액(콤마 포함).

### 6-4. 대납 케이스 (입금자명 ≠ 대여자명)
연체료 목록 비고가 `OOO'으로 입금` / `OOO'로 입금` / `OOO 충당` 형태인 행. 장부 세목(입금자명)과 목록 성명(대여자명)이 다름.

**출력 형식 — 양쪽 이름 모두 표기:**
```
대여자명('입금자명'으로 입금)님 [대여물품] 대여 후 n일 연체되어 연체료 X원 부과
```
예: `최예서('안현서'로 입금)님 쇼핑카트 1개 대여 후 1일 연체되어 연체료 3,000원 부과`

- `으로/로` 조사는 입금자명 받침 유무로 결정(받침 있으면 `으로`, 없으면 `로`). 헷갈리면 목록 비고의 원문 조사를 그대로 사용.
- 필름값·손망실비 케이스도 대납이면 동일하게 `대여자('입금자'으로 입금)님 …` 머리말을 붙임.

**매칭 순서:**
1. 장부 세목 이름 == 연체료목록 성명 → 직접 매칭 (6-3 일반 형식, 머리말 없음).
2. 실패 시: 연체료목록 비고에서 `[장부 이름]'으로 입금` / `[장부 이름] 충당` 검색 → 그 행의 성명을 대여자, 장부 이름을 입금자로 하여 위 형식 사용.
3. 금액·날짜로 교차 검증. 둘 다 모호하면 `검수필요`.

현재 장부 기준 대납 쌍 (장부 입금자 → 목록 대여자):
조소영→서윤아, 이현준→이정빈, 진선우→임성민, 선호진→윤이진, 안현서→최예서, 박우현→권오석

## 7. 예외/검수 정책 (사용자 결정)
- 규칙으로 확정 못 하는 모든 칸 → **공백** 처리.
- 동시에 산출물 어딘가(예: 콘솔 로그 + 각 파일 상단 주석 + 별도 `검수목록.txt`)에 **무엇을/어느 페이지를 비웠는지 명시**.
- 절대 추정해서 채우지 않음.

검수 리포트 항목 예: `[사업3 / 3페이지 / 김연우] 연체료 규정표 없음 → n일 공백`

## 8. 출력 (한글 문서)
- 양식 한글 파일은 Cursor 프로젝트 폴더에 비치(`templates/양식.hwpx` 권장).
- **채우기 방식(권장)**: 양식을 한글에서 **.hwpx**로 저장 → 빈칸에 치환 토큰(`{{사업번호}}`, `{{사업명}}`, `{{날짜}}`, `{{구분}}`, `{{내용}}`, `{{비고}}` 등) 삽입 → 세목/단가는 **행 반복 영역**으로 표시(`{{#세목}}…{{/세목}}`). 코드가 .hwpx(=ZIP+XML)를 풀어 토큰 치환·행 복제 후 재압축.
  - 이유: .hwp(바이너리)는 코드로 직접 쓰기가 불안정. .hwpx는 ZIP/XML이라 안정적.
  - 최종 .hwp가 꼭 필요하면 한글에서 일괄 "다른 이름으로 저장→.hwp", 또는 Windows+한글 환경에서 COM 자동화(`pyhwpx`).
- **파일 분리**: 사업번호(=사업명)별 1개. 파일명 예: `2026-1학기_2_1학기대면사물함배부사업.hwpx`.
- 한 파일 안에서 페이지 = 양식 1장 반복.

## 10. 처리 순서 (main 파이프라인)
1. `config/학기설정.json` + `input/` 엑셀 3개 로드.
2. 장부를 사업번호별로 그룹 → 페이지 분할.
3. 각 페이지: 장부 구분 반영 → 각 칸 채움 → 비고 생성(연체료/환불자 매칭).
4. 매칭/규칙 실패는 공백 + 검수 리포트에 누적.
5. 사업명별 .hwpx 출력 + `검수목록.txt` 생성.
